# PromptVerse Elite Content Generation System

PromptVerse should behave like a curated premium marketplace, not a bulk prompt dump.
This system is outcome-first, use-case driven, and designed to make every prompt feel authored by a specialist.

## Core principles

- Do not mass-produce prompts from interchangeable sentence shells.
- Do not rely on repetitive openers like "Act as...", "Create...", or "Generate..." across the library.
- Do not ship multiple prompts that solve the same problem with small wording changes.
- Every prompt must feel handcrafted, commercially useful, and worth paying for.

## Outcome-first content model

Content planning starts with the user's desired outcome, not with a broad category label.
Examples of acceptable outcomes:

- Professional LinkedIn headshot.
- Luxury watch launch key visual.
- Restaurant promotion campaign.
- Wedding photography hero shot.
- Real estate listing showcase.
- Travel documentary opening sequence.
- Podcast branding package.
- Startup investor pitch assistant.
- Cybersecurity incident analysis workflow.
- SEO content planning system.

Each prompt must map to a specific outcome, audience, and business or creative context.

## Image distribution targets

Across all image prompts, use this target mix:

- Realistic Photography: 40%
- Commercial Advertising: 20%
- Portraits: 10%
- Travel: 10%
- Product Photography: 10%
- Architecture: 5%
- Nature: 3%
- Fantasy: 1%
- Anime: 1% maximum

The image library should overwhelmingly reflect photographers, agencies, brand teams, and commercial creators rather than anime hobby content.

## Reference-image support

Reference-aware prompt families are required for modern image workflows.
Supported placeholders:

- [USER_PHOTO]
- [FACE_REFERENCE]
- [PRODUCT_REFERENCE]
- [STYLE_REFERENCE]
- [POSE_REFERENCE]
- [COMPOSITION_REFERENCE]
- [CHARACTER_REFERENCE]
- [BRAND_REFERENCE]

Any prompt using these placeholders must be tagged as a reference workflow and mapped only to tools that support image conditioning.

## Category voice system

Prompts should sound like the expert who would actually write them:

- Photography prompts: photographer or photo editor voice.
- Advertising prompts: creative director or brand strategist voice.
- Movie prompts: film director or storyboard supervisor voice.
- Music prompts: composer, producer, or sound designer voice.
- Coding prompts: software architect or staff engineer voice.
- Cybersecurity prompts: incident responder or security consultant voice.
- Business prompts: strategy advisor or operator voice.

## Diversity engine

Across every 100 prompts, the library should cover a wide industry spread:

Technology, healthcare, education, finance, travel, luxury brands, restaurants, fitness, sports, architecture, automotive, movies, music, photography, fashion, real estate, artificial intelligence, cloud computing, cybersecurity, programming, e-commerce, podcasting, YouTube, content creation, history, science, space, nature, wildlife, psychology, self improvement, legal, human resources, startups, and marketing.

No single industry should dominate the collection.

## Marketplace quality rubric

Before a prompt is approved, verify:

1. Is it solving a real problem?
2. Does it feel handcrafted rather than generated?
3. Is it structurally different from existing prompts?
4. Would a user reasonably pay for it?
5. Does it provide immediate practical value?
6. Does it sound like a domain specialist wrote it?

If any answer is no, the prompt should be rewritten or rejected.
